# Compliance Crosswalk: ISO 27001 to EU AI Act
**Focus:** Mapping Information Security Controls to AI Regulatory Requirements  
**Framework Version:** 1.0 (2026)

---

### **Overview**
Organizations already compliant with ISO/IEC 27001:2022 have a significant head start in meeting the requirements of the **EU AI Act**. This document identifies the synergy between existing security controls and new AI-specific obligations for high-risk systems.

### **Mapping Table**

| ISO 27001:2022 Control | EU AI Act Requirement | Operational Implementation |
| :--- | :--- | :--- |
| **A.5.1:** Policies for Information Security | **Article 17:** Quality Management System | Integrating AI ethics and model governance into existing corporate security policies. |
| **A.8.2:** Privileged Access Rights | **Article 16(d):** Technical Documentation | Restricting model tuning and weights access to authorized personnel; maintaining detailed logs of all modifications. |
| **A.8.16:** Monitoring Activities | **Article 61:** Post-Market Monitoring | Continuous monitoring for model drift, hallucination rates, and security vulnerabilities after deployment. |
| **A.5.23:** Information Security for Supplier Relationships | **Article 28:** Responsibilities of Distributors/Importers | Assessing third-party AI providers (e.g., OpenAI, Anthropic) for data handling and compliance certification. |

### **Conclusion**
By leveraging the existing ISO 27001 ISMS (Information Security Management System), the firm can achieve approximately **60% readiness** for EU AI Act compliance through strategic documentation updates rather than complete infrastructure overhauls.
